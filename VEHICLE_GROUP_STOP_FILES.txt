API SUMMARY
===========

Main Function:
  Vehicle::setSequentialStop(vehIDs, edgeID, pos, ...)
  - Stop multiple vehicles sequentially at same position
  
Query Functions:
  Vehicle::getStopSequence(edgeID, pos)
  - Get list of vehicles at a stop
  
  Vehicle::getSequentialStopPosition(vehID, edgeID, basePos, ...)
  - Get calculated position for vehicle in sequence
  
Cleanup Function:
  Vehicle::clearStopSequence(edgeID, pos, ...)
  - Remove stop sequence information

KEY FEATURES
============

✓ Sequential queue formation at same position
✓ Automatic position calculation for each vehicle
✓ Configurable spacing between vehicles (2.5m default)
✓ Upstream edge overflow for overflow vehicles
✓ Support for all stop parameters (duration, flags, etc.)
✓ Error handling and validation
✓ Backward compatible with existing code
✓ Zero performance impact if not used

TROUBLESHOOTING
===============

If API methods not found:
1. Rebuild SUMO after pulling latest changes
2. Verify VehicleGroupStop.cpp/.h are in src/libsumo/
3. Check CMakeLists.txt includes them in build

If position calculations seem wrong:
1. Verify vehicle lengths in vehicle type definitions
2. Check spacing parameter value
3. See VEHICLE_GROUP_STOP_QUICK_REFERENCE.md examples

If overflow not working:
1. Ensure allowUpstreamOverflow=true
2. Check upstream edges exist and are connected
3. Verify no route constraints block access

For more help: See VEHICLE_GROUP_STOP_README.md troubleshooting section
